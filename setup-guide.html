Before You Start
What You Need
This guide walks you through setting up your cold email automation app from scratch. You will not use the terminal or command line at all. Everything is done through websites you already know how to use.
You need three things before starting:
1
A Google account (Gmail works perfectly)
2
Your GitHub repos (already created for you):
• Frontend: github.com/interiorvisualsd-maker/cold-outreach-frontend
• Backend: github.com/interiorvisualsd-maker/cold-outreach-backend
3
30–45 minutes of focused time
💡 How to think about it: You're setting up three websites that work together. Think of it like a restaurant — Neon is the pantry (stores data), Google Cloud is the kitchen (does the work), and Vercel is the dining room (where you sit and interact).
Step 1 · 5 minutes
Create Your Database (Neon)
Neon stores all your data — leads, emails, replies, and accounts
1
Open https://neon.tech in your browser
2
Click "Sign up" (top right corner)
3
Click "Continue with Google" and pick your Google account
4
On the "Create your first project" page:
• Project name: type lead-dispatcher
• Database name: leave as neondb
• Region: pick the closest to you (if unsure, pick US East (Ohio))
• Postgres version: leave default
5
Click "Create project"
6
You'll see a page with a long connection string that looks like:
postgresql://lead_dispatcher_owner:AbCdEf@ep-cool-name.us-east-2.aws.neon.tech/neondb?sslmode=require
⚠️ IMPORTANT: Copy this ENTIRE link and save it somewhere safe (Notes app, password manager, etc.). You'll need it 2 more times. This is your DATABASE LINK.
Step 2 · 10 minutes
Set Up Google Cloud
Google Cloud runs your backend — the "brain" that sends emails and checks replies
2A. Create a Google Cloud project
1
Open https://console.cloud.google.com and sign in with your Google account
2
At the top, click the project dropdown (says "Select a project")
3
Click "New Project" (top right of popup)
4
Project name: type lead-dispatcher
5
Click "Create"
6
Wait 10 seconds, then click the bell icon (🔔 top right) → "Select project"
✅ You automatically get $300 free credit valid for 90 days. You won't be charged anything.
2B. Turn on 4 services
For each one: search the name → click the result → click "Enable"
1
Search Cloud Run API → Enable
2
Search Cloud Build API → Enable
3
Search Cloud Scheduler API → Enable
4
Search Artifact Registry API → Enable
2C. Generate 3 secret keys
Your app needs secret keys to keep passwords and data safe. Use free web tools to generate them:
CRON SECRET (for the email worker):
1
Go to https://www.random.org/strings
2
Set: Length = 32, Strings = 1, Character set = Alphanumeric
3
Click "Get Strings" → copy the result
JWT SECRET + ENCRYPTION KEY (for passwords):
1
Go to https://www.allkeysgenerator.com/Random/Security-Encryption-Key-Generator.aspx
2
Set key size to 256-bit and click "Generate"
3
Copy the hex string — this is your JWT SECRET
4
Click Generate again → copy — this is your ENCRYPTION KEY
⚠️ Save all 3 secrets somewhere safe. You'll need them in the next step.

Step 4 · 5 minutes
Set Up the Email Worker
Makes your app check every 5 minutes for emails to send and replies to process
4A. Create a Cloud Scheduler job
1
In Google Cloud Console, search Cloud Scheduler → click it
2
Click "Create Job"
3
Fill in:
• Name: lead-dispatcher-worker
• Description: Sends emails and checks replies every 5 minutes
• Frequency: type */5 * * * * (this means "every 5 minutes")
• Timezone: pick your timezone
4
Click "Continue"
5
Under "Configure the execution":
• Target type: select "HTTP"
• URL: type your backend URL + /api/cron/ + your CRON SECRET
  Example: https://cold-outreach-api-1234567-uc.a.run.app/api/cron/AbCdEf123456
  (Replace with your actual BACKEND URL and CRON SECRET)
• HTTP method: POST
• Headers: click "Add header" → Key: Content-Type, Value: application/json
• Body: type {}
6
Click "Create"
4B. Test it
1
You'll see your job in the list
2
Click the three dots (⋮) on the right → "Run now"
3
Wait 10 seconds, then check "Last run result" — should say "Success" ✅
💡 If it fails, check: Is the URL correct? (should end with /api/cron/YOUR_SECRET). Is the method POST? Did you include the CRON_SECRET in the URL?

Troubleshooting
Common Issues & Fixes
🔧 "Failed to fetch" or blank page
Your website can't reach your backend. Check:
1. In Vercel: is NEXT_PUBLIC_API_URL set to your BACKEND URL?
2. In Cloud Run: is FRONTEND_URL set to your Vercel URL?
3. Did you redeploy after changing them?
🔧 "Unauthorized" errors
Your login expired. Just log out and log back in.
🔧 Database won't connect
Your Neon link needs ?sslmode=require at the end:
postgresql://user:pass@host/db?sslmode=require
🔧 Cloud Run deployment fails
1. Make sure all 4 APIs are enabled (Step 2B)
2. Make sure you selected "Allow unauthenticated invocations"
3. Make sure your GitHub repo is connected properly
🔧 Worker doesn't seem to run
1. Go to Google Cloud Console → Cloud Scheduler
2. Click your lead-dispatcher-worker job
3. Click "Run now" to test
4. Check the logs for errors
5. Verify the URL is correct: https://YOUR-BACKEND-URL/api/cron/YOUR-CRON-SECRET
🔧 Email sending fails
1. Gmail: You MUST use an App Password, not your regular password
2. Daily cap too high: Start with 50, not 100
3. Account needs warm-up: New accounts should warm up for 1–2 weeks
